# NPlusOneIncident

Represents a detected N+1 query issue in Entity Framework Core, capturing diagnostic details such as the executed SQL, row count, severity, and call stack for analysis or reporting.

## API

### `public string SqlQuery`

The raw SQL query text executed by Entity Framework Core that triggered the N+1 incident. This value is populated from the diagnostics infrastructure during query execution.

### `public int Count`

The number of rows returned by the SQL query. This count is used to determine the magnitude of the N+1 issue and is compared against expected thresholds to assess severity.

### `public NPlusOneSeverity Severity`

Gets the severity level of the incident, indicating the impact of the N+1 query. Higher severities suggest more critical performance issues. This value is derived from the `Count` and configured thresholds.

### `public string StackTrace`

The call stack at the point where the N+1 query was detected. This trace is captured during diagnostics and is useful for pinpointing the source of the inefficient query in application code.

### `public string Generate()`

Generates a human-readable summary of the incident, including the SQL query, row count, severity, and stack trace. The output format is suitable for logging or display in diagnostic tools.

**Returns**
A formatted string containing all incident details.

**Throws**
`InvalidOperationException` if required properties (`SqlQuery`, `Count`, or `Severity`) are not set before calling this method.

### `public void WriteToFile(string filePath)`

Writes the incident details to a file at the specified path. The file is overwritten if it already exists. The output includes the same details as generated by `Generate()`.

**Parameters**
- `filePath` (string): The absolute or relative path to the output file. Must not be null or empty.

**Throws**
- `ArgumentNullException` if `filePath` is null.
- `ArgumentException` if `filePath` is empty or contains invalid characters.
- `UnauthorizedAccessException` if the caller lacks permission to write to the file.
- `DirectoryNotFoundException` if the parent directory does not exist.
- `IOException` if the file cannot be written for any other reason.
